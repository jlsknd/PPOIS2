# Лабораторная работа №2
## Проектирование программного обеспечения интеллектуальных систем 

### Складской учёт - приложение для управления товарами на складе


## 1. Цель работы

Изучить принципы построения графического пользовательского интерфейса и разработать программную систему с графическим пользовательским интерфейсом на языке Python с использованием архитектурного шаблона Model-View-Controller (MVC).


## 2. Задание

Разработать оконное приложение для управления складскими запасами с возможностью:
- Добавления, поиска, удаления записей
- Поиска по трём вариантам критериев
- Постраничного вывода данных (пагинация)
- Двух видов отображения: табличный и древовидный
- Хранения данных в SQLite базе данных
- Импорта/экспорта в XML (DOM для записи, SAX для чтения)
- Сохранения/загрузки в файл базы данных


## 3. Используемые технологии

| Технология | Назначение |
|------------|------------|
| **Python 3.8+** | Язык программирования |
| **PyQt5** | Библиотека для создания графического интерфейса |
| **SQLite3** | Встроенная база данных для хранения записей |
| **xml.dom.minidom** | DOM парсер для записи XML |
| **xml.sax** | SAX парсер для чтения XML |



## 4. Архитектура приложения

### 4.1 Общая архитектура

Приложение построено по архитектурному шаблону **Model-View-Controller (MVC)**.


### 4.2 Тип модели

В приложении реализован **гибридный подход**:

- **Пассивная модель** — между View и Model (View не знает о Model, Model не знает о View)
- **Активный контроллер** — Controller активно уведомляет View об изменениях через сигналы PyQt5


### 4.3 Структура проекта

```
warehouse_app/
│
```

## 5. Реализация компонентов

### 5.1 Модель (Model)

#### 5.1.1 Сущность Product (model/product.py)

```python
@dataclass
class Product:
    """Продукт"""
    id: Optional[int] = None      # ID в БД
    name: str = ""                 # Название товара
    manufacturer: str = ""         # Производитель
    unp: str = ""                  # УНП производителя
    quantity: int = 0              # Количество на складе
    address: str = ""              # Адрес склада
```

**Методы:**
- `to_dict()` — преобразование в словарь для передачи в View
- `from_dict()` — создание объекта из словаря

## 6. Форматы данных

### 6.1 XML формат

```xml
<?xml version="1.0" encoding="utf-8"?>
<products>
  <product id="1">
    <name>Ноутбук Lenovo ThinkPad E15</name>
    <manufacturer>ООО Марвел-Маркет</manufacturer>
    <unp>192040812</unp>
    <quantity>15</quantity>
    <address>г. Минск, ул. Тимирязева, 65Б</address>
  </product>
</products>
```

### 6.2 База данных SQLite

**Таблица products:**

| Поле | Тип | Ограничение |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT |
| name | TEXT | NOT NULL |
| manufacturer | TEXT | NOT NULL |
| unp | TEXT | NOT NULL |
| quantity | INTEGER | NOT NULL |
| address | TEXT | NOT NULL |
